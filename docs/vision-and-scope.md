# Vision and Scope

## Phase 1: Internal Experiment (Current)

Phase 1 is a focused, internal pilot with the Glossa team working in Malayalam to adapt Flora, Fauna, and Realia resources. The goal is to prove that Glossa can replace disconnected tools (Google Sheets, email, manual validation) with a single, reliable system that combines:

- Grounded source material (English reference resources)
- AI-assisted adaptation (not generation)
- Structural validation (ensuring Bible refs, cross-refs, and metadata stay intact)
- Team collaboration with clear roles
- Immediate publication of completed work

**Timeline:** 15 days to a working basic system

## Why This Matters

Translation teams working with structured biblical content face recurring problems:

- Manual collaboration tools (spreadsheets, email) lack workflow clarity
- Metadata and structural elements are fragile and easy to corrupt
- Quality checks arrive too late to prevent rework
- No visibility into team progress or entry status
- Publishing is manual and error-prone

Phase 1 tests whether Glossa can solve these problems for one team, in one language, with three resource domains.

## Who This Phase 1 Serves

- **Translators** — Edit entries, see validation feedback, submit for review
- **Reviewers** — Check accuracy, approve ready work, handle validation exceptions
- **Team Admin** — Manage workflow settings and see complete audit history
- **Public readers** — Access completed resources in Malayalam, with English fallback for untranslated entries

## Phase 1 Core Capabilities

- **Resource import** — Import English Flora/Fauna/Realia into a structured database
- **Entry editor** — Translators draft adaptations with AI assistance, edit blocks freely, save work
- **Deterministic validation** — Check that Bible refs, cross-refs, protected tokens, and metadata are preserved
- **Bypass recording** — All validation failures logged with reason, approver, and timestamp
- **Workflow states** — Draft → Ready (publishing trigger) → Public
- **Public site** — Localized entries published immediately when marked Ready; untranslated entries show English fallback
- **Reader feedback** — Ratings, comments, and flags on public entries
- **RBAC** — Translator, Reviewer, Admin roles with clear authority paths
- **Audit trail** — Complete history of all edits, validations, bypasses, and approvals

## Phase 1 Scope Boundaries

**In scope for Phase 1:**
- One team, one language (Malayalam)
- Three resource domains (Flora, Fauna, Realia)
- Core workflow end-to-end
- Structural integrity enforcement
- Team-controlled publishing pipeline
- Basic public site

**Deferred to Phase 2+:**
- Multi-language team scaling
- BibleBrains integration
- Advanced search and filtering
- Analytics and reporting dashboards
- External organization onboarding
- Self-serve platform features

## Success Criteria for Phase 1

Phase 1 is successful if:
- Core workflow runs end-to-end without manual workarounds
- Structural validation catches real issues consistently
- AI-assisted drafting produces usable output for translators to improve
- Malayalam team confirms Glossa is better than Google Sheets
- Public site displays completed entries correctly with proper fallback behavior
- All bypasses are auditable and traceable
